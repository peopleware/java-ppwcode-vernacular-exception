The PeopleWare ContractTaglets Module is a module that allows the user to create his own handlers for custom contract taglets in java source code.

At this moment, the following contract taglets are used:

  • invar
  • basic
  • pre
  • post
  • result

The module contains one abstract generic class:

  • {@link be.peopleware.taglet.contract.AbstractContractTaglet}

Abstract class for run-time registration of taglets in this package:

  • {@link be.peopleware.taglet.contract.Registrar}

And five concrete classes - one for each taglet:

  • {@link be.peopleware.taglet.contract.BasicTaglet}
  • {@link be.peopleware.taglet.contract.InvarTaglet}
  • {@link be.peopleware.taglet.contract.PostTaglet}
  • {@link be.peopleware.taglet.contract.PreTaglet}
  • {@link be.peopleware.taglet.contract.ResultTaglet}

To create new contract taglet you should, on top of other things you do when you create custom taglet, add few lines to (default) constructor.

For example, constructor in the class {@link be.peopleware.taglet.contract.ResultTaglet} have the following additional lines of code: 
	$allowedKeywords.add(AbstractContractTaglet.KEYWORD_RESULT);
    $allowedKeywords.add(AbstractContractTaglet.KEYWORD_NEW);
    $allowedKeywords.add(AbstractContractTaglet.KEYWORD_FORALL);
By doing this we allow using following in our taglet result, new and forall.
